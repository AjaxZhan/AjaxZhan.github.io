CPlusPlus Notes(I)：内存分区和OOP

发表于2023-03-08|更新于2023-03-14|C++
|字数总计:5.4k|阅读时长:23分钟|阅读量:|评论数:

本笔记大致记录了C++的基础内容，最近的更新是将所以的demo都重新设计并自己敲了一遍，增强对知识点的理解。

C++的内存分区模型

内存分区

  • 内存分区：
    • (程序运行前)代码区：二进制代码。操作系统管理。
      • 存放CPU执行的机器指令。
      • 共享的（对频繁运行的程序，只需要内存的一份代码）
      • 只读的（防止程序意外修改其指令）
    • (程序运行前)全局区：全局变量、静态变量、常量（字符串常量、其它常量）
    • (程序运行后)栈区：局部变量、函数参数等。由编译器自动分配和释放
      • 不要返回局部变量地址。
    • (程序运行后)堆区：由程序员分配和释放。
      • C++用new在堆区开辟内存。

new操作符

  • 语法：
    • int *p = new int(10)
    • delete p;
  • 开辟数组：
    • int *arr = new int[10];
    • delete[] arr;

引用

  • 基本含义：起别名。
  • 注意：
    • 必须初始化！错误：int &c;
    • 初始化后不能改变。当然，可以赋值，赋值不是改变引用。
  • 引用做函数参数：和地址传递效果相同。但是语法更简单。
  • 引用做函数返回值：
    • 首先，不要返回局部变量！
    • 用法：函数可以做左值。q.front() = 1000; test02() = 1000;
  • 引用的本质：
    • 引用本质上是一个指针常量。所有指针操作由编译器帮我们做。
    • int& ref -- int* const ref = &a;
  • 常量引用：
    • 作用：修饰形参，防止误操作。
    • void showValue(const int &v)
  • 小练习
    • int &ref = 100 ; 错误，需要合法的内存空间。
    • const int& ref = 10; 正确， 编译器会优化相当于int tmp = 10; const int &ref = tmp;

函数提高篇

默认参数

  • 如果某个位置有默认值，后面必须都要有默认值。
  • 函数声明有默认值，实现的时候不能有默认值。

占位参数

  • 占位参数必须填补！

函数重载

  • 含义：函数名可以重复，提高复用性。
  • 条件：
    • 同作用域！
    • 函数名相同
    • 函数参数类型不同、个数不同或者顺序不同
  • 特别注意：
    • 返回值不能作为函数重载的条件。
    • 函数重载碰到默认参数：有歧义，应该避免。
      • void func(int a, int b = 10); void func(int a); func(10);
      • 这样就不知道应该调用哪个了。
    • 函数重载碰到引用
      • void func(int &a); void func(const int &a);
      • func(10);调用后者
      • int a =10 ; func(a); 调用前者

类和对象

OOP思想我认为Java语言体现得最佳，因此此处主要讲语法，不讲思想。

权限

  • public 类内访问，类外也可以访问
  • protected 类内访问，类外不可以访问
  • private 类内访问，类外不可以访问

引出：set、get方法。

struct和class的区别

  • 唯一区别：默认访问权限不同。
    • class：默认权限为private
    • struct：默认权限为public

对象初始化和清理

构造函数和析构函数

  • 编译器会自动调用，如果不提供这两个函数，编译器会提供。
  • 编译器提供的构造函数和析构函数是空实现的。
  • 语法：
    • 构造函数：无返回值；类名为函数名；可以重载；程序自动调用。
    • 析构函数：无返回值；类名为~函数名；不能重载；程序自动调用
    • Person(){} 和 ~Person(){}
  • 分类：
    • 有参构造和无参构造。
    • 拷贝构造和普通构造。
    • 调用方式：括号、显式、隐式转换。

拷贝构造函数调用时机

  • 使用一个已经创建完毕的对象来初始化一个新对象。
  • 值传递方式给函数参数传值
  • 以值方式返回局部对象

构造函数调用规则

  • 编译器会给一个类添加三个函数
    • 默认构造
    • 默认析构
    • 默认拷贝，对属性进行赋值。
  • 如果用户提供有参构造，CPP不提供无参，但是提供拷贝。
  • 如果用户定义拷贝，CPP不提供任何构造。

深拷贝和浅拷贝

  • 浅拷贝：简单的赋值操作。
  • 深拷贝：重新在堆区申请内存。
  • 如果属性中有在堆区开辟的，一定要自己提供拷贝构造函数，防止浅拷贝问题。

总的来说，这块需要注意的点还挺多的。比如释放内存的时候防止空指针异常，拷贝构造时候需要取值运算符。

初始化列表

  • C++提供了初始化列表语法，用来初始化属性。
  • 本来感觉很少用，但是好像很多demo都在用。

类对象作为类成员

有点像Java的内部类？好吧不是，内部类是在类声明内声明类。

  • C++类中的成员可以是另一个类的对象。
  • 这里不妨令B中有一个A，我们称A为对象成员。
  • 那么问题来了：A与B的构造函数和析构函数顺序是什么呢？
    • ANS：先调用A的构造，再调用B构造。析构相反，先调用B的析构，再析构A。

静态成员

  • 静态成员：static关键字，跟Java的类变量是一样的。
  • 分类
    • 静态成员变量
      • 所有对象共享一份数据。
      • 编译阶段分配内存。
      • 类内声明，类外初始化。
    • 静态成员函数
      • 所有对象共享通过一个函数。
      • 静态成员函数只能访问静态成员变量。

C++对象模型和this指针

成员变量和成员函数分开存储

  • C++成员变量和成员函数是分开存储的，这意味着多个同类型的对象会公用一块代码。

this指针

  • 由于C++的非静态成员函数只会诞生一份函数实例，这一块代码如何区分哪个对象调用自己就成了问题。
  • this指针指向被调用的成员函数所属的对象。
  • this指针是隐含每一个非静态成员函数内的一种指针。
  • 用途：
    • 当形参和成员变量同名时，可以用this函数区分。
    • 在类的非静态成员函数返回对象本身，可以用return *this
  • this指针的本质：指针常量。指针指向谁不能修改。

空指针访问成员函数

  • C++中空指针也可以调用成员函数。
  • 要注意有没有用到this指针，如果用到了this指针，需要加一个判断保证鲁棒性。

const修饰成员函数

  • 常函数：成员函数加const，称之为常函数。比如重载运算符的时候。
    • void showPerson() const{}
    • 常函数：不可以修改成员属性。
    • this指针本身是一个指针常量，指向谁是不能修改的，如果想要让指向的值本身也不能修改，就需要这个常函数。
    • 成员属性声明加上mutable 后，在常函数中仍然可以修改。（mutable的中文意思是用情不专的；易变的）
  • 常对象：
    • 声明对象前加const。
    • const Person person;
    • 常对象只能调用常函数。

友元

  • 作用：有些私有属性需要允许类外的特殊函数或者特殊的类进行访问，就需要用到友元的技术。
  • 简而言之，友元的目的就是让一个函数或类访问另一个类的私有成员。
  • 语法关键字：friend
  • 三种实现方式：
    • 全局函数做友元
    • 类做友元
    • 成员函数做友元

OOP之继承

  • 继承的好处：减少重复代码。
  • 实现语法：class A : public B A 类为子类（派生类） B为父类（基类）
  • 派生类中的成员
    • 从基类继承过来的。 — 体现其共性。
    • 新增的成员。 — 体现其个性。
  • 继承的三种方式
    • 公有继承：将非私有属性继承并赋为public属性。
    • 保护继承：将非私有属性继承为protect属性。
    • 私有继承：将非私有属性继承为private属性。可以访问，但是原来的private属性无法访问。

继承中的对象模型

  • 问题：从父类继承过来的成员，哪些属于子类对象？
  • ANS：实际上，私有成员会被继承，只是被隐藏。

继承中的构造和析构顺序

  • 结论：先构造父类，再构造子类。
  • 先析构子类，再析构父类。

继承同名成员处理方式

  • 需求：当子类和父类出现同名成员，子类如何区分呢？
  • 结论：
    • 访问子类同名成员：直接访问。son.func();
    • 访问父类同名成员：加作用域。（Java是super）son.Base::func();

同名静态成员处理方式

  • 需求：继承了同名的静态成员在子类如何访问？
  • 结论：访问子类同名：直接访问；访问父类同名：需要加作用域。
    • 类名访问属性：Son::m_A 、 Son::Base::m_A;
    • 类名访问成员函数：Son::func(); \ Son::Base::func();

多继承语法（Java是单继承）

  • 语法：class 子类 : 继承方式 父类1， 继承方式 父类2， 继承方式 父类3 ...
  • 问题：可能会引发同名成员的问题，需要加作用域区分。
  • C++开发中不建议用多继承。

菱形继承

  • 概念：两个派生类同时继承一个基类，又同时被某一个类继承。
  • 比如：🐏和驼继承动物，并且都被羊驼继承。
  • 问题：
    • 🐏继承了动物的数据，驼同样继承了动物的数据，草泥马使用数据的时候就会产生二义性。
    • 实际上，我们只需要一份动物的数据即可。
  • 解决：虚继承技术。

多态

  • 多态分类：
    • 静态多态
      • 函数重载；运算符重载
      • 静态多态函数地址早绑定：编译阶段确定函数地址。
    • 动态多态
      • 派生类、虚函数
      • 动态多态的函数地址晚绑定：运行阶段确定函数地址。
  • 多态的满足条件
    • 存在继承关系。
    • 子类重写父类中的虚函数。
  • 多态的使用条件
    • 父类指针或者父类引用指向子类对象。

纯虚函数和抽象类

  • 从上面的例子可以看到，父类虚函数的实现实际上没什么意义。主要都是子类重写的方法。
  • 因此可以将虚函数改为纯虚函数。virtual 返回值 函数名( 列表) = 0;
  • 类中有了纯虚函数，这个类叫做抽象类。
  • 抽象类的特点
    • 无法实例化对象。
    • 子类必须重写父类的纯虚函数，否则也属于抽象类。
