C++学习笔记(一)：内存分区、OOP、文件

发表于2023-03-08|更新于2023-03-08|C++
|字数总计:1.6k|阅读时长:5分钟|阅读量:|评论数:

本专题记录上学期黑马程序员C++学习笔记，谨以此作为复习

C++的内存分区模型

内存分区

  • 内存分区：
    • (程序运行前)代码区：二进制代码。操作系统管理。
      • 存放CPU执行的机器指令。
      • 共享的（对频繁运行的程序，只需要内存的一份代码）
      • 只读的（防止程序意外修改其指令）
    • (程序运行前)全局区：全局变量、静态变量、常量（字符串常量、其它常量）
    • (程序运行后)栈区：局部变量、函数参数等。由编译器自动分配和释放
      • 不要返回局部变量地址。
    • (程序运行后)堆区：由程序员分配和释放。
      • C++用new在堆区开辟内存。

new操作符

  • 语法：
    • int *p = new int(10)
    • delete p;
  • 开辟数组：
    • int *arr = new int[10];
    • delete[] arr;

引用

  • 基本含义：起别名。
  • 注意：
    • 必须初始化！错误：int &c;
    • 初始化后不能改变。当然，可以赋值，赋值不是改变引用。
  • 引用做函数参数：和地址传递效果相同。但是语法更简单。
  • 引用做函数返回值：
    • 首先，不要返回局部变量！
    • 用法：函数可以做左值。q.front() = 1000; test02() = 1000;
  • 引用的本质：
    • 引用本质上是一个指针常量。所有指针操作由编译器帮我们做。
    • int& ref -- int* const ref = &a;
  • 常量引用：
    • 作用：修饰形参，防止误操作。
    • void showValue(const int &v)
  • 小练习
    • int &ref = 100 ; 错误，需要合法的内存空间。
    • const int& ref = 10; 正确， 编译器会优化相当于int tmp = 10; const int &ref = tmp;

函数提高篇

默认参数

  • 如果某个位置有默认值，后面必须都要有默认值。
  • 函数声明有默认值，实现的时候不能有默认值。

占位参数

  • 占位参数必须填补！

函数重载

  • 含义：函数名可以重复，提高复用性。
  • 条件：
    • 同作用域！
    • 函数名相同
    • 函数参数类型不同、个数不同或者顺序不同
  • 特别注意：
    • 返回值不能作为函数重载的条件。
    • 函数重载碰到默认参数：有歧义，应该避免。
      • void func(int a, int b = 10); void func(int a); func(10);
      • 这样就不知道应该调用哪个了。
    • 函数重载碰到引用
      • void func(int &a); void func(const int &a);
      • func(10);调用后者
      • int a =10 ; func(a); 调用前者

类和对象

OOP思想我认为Java语言体现得最佳，因此此处主要讲语法，不讲思想。

权限

  • public 类内访问，类外也可以访问
  • protected 类内访问，类外不可以访问
  • private 类内访问，类外不可以访问

引出：set、get方法。

struct和class的区别

  • 唯一区别：默认访问权限不同。
    • class：默认权限为private
    • struct：默认权限为public

对象初始化和清理

  • 为什么？安全问题。

构造函数和析构函数

  • 如果不提供构造器和析构函数，编译器会提供。
    • 编译器提供的构造和析构是空实现。
  • 构造函数：
    • 用于创建对象时，对成员属性赋值，由编译器自动调用。
    • 语法：
      • 没有返回值，也不写void。
      • 函数名和类名相同。
      • 构造函数可以重载。
      • 构造函数只会调用一次。
  • 析构函数；
    • 用于在对象销毁前系统自动调用，执行清理工作。
    • 语法：
      • 函数名和类名相同。
      • 不可以有参数，不可以重载。

构造函数的分类及调用

语法挺繁杂的。

  • 分类方式
    • 参数：有参构造、无参构造
    • 类型：普通构造、拷贝构造
  • 调用方式
    • 括号法
    • 显示法
    • 隐式转换法

拷贝构造的调用时机

这块emmm

  • 时机：
    • 使用已经创建完毕的对象来初始化一个新对象。
    • 值传递的方式给函数参数传值。
    • 以值方式返回局部对象。

构造函数调用规则

  • 编译器默认给一个类添加的函数。

    • 默认构造函数。无参，函数体为空。
    • 默认析构函数。无参，函数体为空。
    • 默认拷贝构造函数，对属性进行值拷贝。

  • 构造函数调用规则

    • 如果用户有有参构造，C++不会提供无参，但是会提供拷贝构造。
    • 如果用户定义拷贝构造，C++不会再提供其它构造函数。

  • 如果不写拷贝构造，编译器会自动添加拷贝构造，并且做浅拷贝。

  • 深拷贝和浅拷贝
    • 浅拷贝：简单的赋值拷贝。
    • 深拷贝：堆区重新申请内存，进行拷贝操作。
  • 如果属性在堆区开辟，一定要自己提供拷贝构造，防止浅拷贝。
