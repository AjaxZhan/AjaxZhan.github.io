JavaSE--IO流

发表于2023-02-11|更新于2023-02-11|JAVA后端
|字数总计:3.5k|阅读时长:13分钟|阅读量:|评论数:

文件

什么是文件流

  • 文件流
    • 文件在程序中以流的形式来操作。以内存作为参考对象。
    • 从Java内存写入到磁盘，称之为输出流
    • 从磁盘读入到内存，称之为输入流

常用文件操作

如何创建文件？

  • File实现了串行化接口 + 比较接口
  • 创建文件对象相关构造器和方法
    • new File(String pathname); 根据路径构建一个File对象
    • new File(File parent, String child); 根据父目录文件 + 子路径构建
    • new File(String parent, String child); 根据父目录+ 子路径构建
  • 案例：在D盘的Test目录下创建文件
    • createNewFile方法
    • 基本思路：创建一个File对象，然后执行创建文件的方法。file.createNewFile();

目录操作和文件操作

常见API：

  • file.exists()
  • file.delete()
  • file.mkdirs() //mkdir创建一级目录，s创建多级目录

IO流原理及流的分类

  • IO流用于处理数据传输，如读写数据和网络通讯。
  • java.io 包
  • 输入流：从磁盘读到程序
  • 输出流：从程序输出到磁盘

流的分类

  • 按照数据单位分为：字节流（8 bit）、 字符流（根据编码，以字符为单位）
    • 字节流操作二进制文件，如声音、视频等保证无损。
    • 字符流操作文本文件
  • 按照数据流的方向：输入流、输入流
  • 按照流的角色：节点流、处理流（包装流）
  • 下面四个类都是抽象类。
    • Java的IO流涉及40多个类，都是从4个抽象基类中派生出来的。
    • 名字都是以父类名为后缀。
抽象基类 字节流 字符流
输入流 InputStream Reader
输出流 OutputStream Writer
  • 流和文件的关系？
    • 输入流和输出流，是传输媒介
    • 文件中是数据，是被传输的对象。

字节输入流—InputStream

常用子类

  • FileInputStream 文件字节输入流
  • BufferedInputStream 缓冲字节输入流（包装流）
  • ObjectInputStream 对象字节输入流

FileInputStream

  • content = fileInputStream.read()
    • 从该输入流中读取一个字节的数据，返回数据的下一个字节，如果达到文件末尾返回-1
    • 汉字是按照3个字节来表示，如果一个字节的来读取，必然是出错的。
    • 所以，如果是文本文件，最要用字符流来处理。
  • fileInputStream.close();：读完要关闭流，因为流是一种资源
  • readLen = fileInputStream.read(buf)
    • 如果读取正常，返回读入缓冲区的总字节数.
    • byte[] buf = new byte[8];可以先定义byte数组，一次读取多个，速度比较快。
    • 如何输出读到的东西？System.out.print((new String(buf,0,readLen)))

FileOutputStream

  • new FileOutputStream(filePath) 这种流的创建方式，当写入内容时候会覆盖原来的内容。
  • new FileOutputStream(filePath, boolean append) 这种方式当写入内容时，是追加到文件后面
  • fileOutputStream.write(‘A’);写入一个字节
  • fileOutputStream.write(str.getBytes()); 利用字符串转换为字节数组的方法
  • fileOutputStream.write(str.getBytes(),0,str.length());

小案例：编程完成图片或者音乐文件的拷贝

  • 创建文件的输入流，将E盘文件读取到Java程序
  • 创建文件的输出流，将读取到的文件写入到指定的位置
  • 读取部分数据就写入到指定的文件

FileReader 和 FileWriter 字符流


FileReader

读取其实跟OutputStream差不多，但是用的是char数组。

FileWriter

  • 支持写入单个字符、字符数组、数组的指定部分、字符串、字符串某部分

    • fileWriter.write(‘H’);
    • fileWriter.write(chars);
    • fileWriter.write(“cagurzhan”.toCharArray(),0,3);
    • fileWriter.write(“你好北京 ~”);
    • fileWriter.write(“PKU~~”,0,4);

  • 注意，一定要关闭或者刷新，才能真正将内存中的数据写入到文件

    • finally 那里写上：fileWriter.close()

节点流和处理流

  • 节点流：
    • 从特定数据源读写数据。如FileWriter 和 FileReader FileInputStream FileOutputStream
    • 数据源：存放数据的地方。可以是文件、字符串、数组、管道。
    • 比较低级，底层的流。
  • 处理流（也叫包装流）:
    • 例如，BufferedReader类中，有属性Reader。即可以封装一个节点流。
    • 这个节点流只需要为Reader的子类即可。
    • 连接在已存在的流之上，为程序提供强大的读写功能，而且更加灵活。
    • 体现了修饰者模式
  • 节点流和处理流的区别和联系
    1. 节点流：是底层流，直接跟数据源相连接。
    2. 包装流(处理流)：包装节点流，既可以消除不同节点流的实现差异，也可以提供更方便的方法来完成输入输出。
    3. 包装流对节点流包装，使用修饰器的设计模式，不会直接跟数据源相连。


BufferedReader & BufferWriter 字符流，处理流

  • 字符流，不要操作二进制文件，否则容易造成文件损坏
  • 处理流关闭时，只需要关闭外层流即可。
    • bufferedReader.close();
    • 底层调用了节点流的close方法。
  • BufferedReader
    • BufferedReader bufferedReader = new BufferedReader(new FileReader(filePath));
    • line = bufferedReader.readLine() 返回包含行内容在字符串，读到流末尾则null
  • BufferWriter
    • BufferedWriter bufferedWriter = new BufferedWriter(new FileWriter(filePath,true));
    • bufferedWriter.write(“hello cagur”);
    • bufferedWriter.newLine(); // 插入一个和系统相关的换行符

BufferedInputStream & BufferedOutputStream 字节流

  • 是FilterInputStream的直接子类，也是InputStream的子类

对象处理流 ObjectInputStream & OOS

  • 需求：
    • 将一个数据保存到文件，并且能从文件中恢复数据类型。
  • 序列化和反序列化
    • 序列化：保存数据时，保存数据的值和数据类型。
    • 反序列化：恢复数据时，恢复数据的值和数据类型。
    • 需要让某个对象支持序列化机制，则必须让其类时可序列化的。实现如下两个接口之一：
      • Serializable 标记接口（声明机制，没有方法）
      • Externalizable（继承Serializable，有方法需要实现，一般不用这个）

  • OIS 和 OOS

    • 提供了基本数据类型和引用类型的序列化和反序列化
    • OOS 提供序列化
    • OIS 提供反序列化
    • 接收InputStream

  • OOS序列化

    • oos.write(100); // int自动装箱成integer，而Integer是实现了序列化接口
    • oos.writeBoolean(true);
    • oos.writeChar(‘a’);
    • oos.writeDouble(9.5);
    • oos.writeUTF(“ajaxzhan”);
    • oos.writeObject(new Dog(“wa”,10));
    • oos.close();
  • 注意：反序列化的顺序必须和序列化的顺序保持一致
  • OIS反序列化
    • System.out.println(ois.readInt());
    • System.out.println(ois.readBoolean());
    • System.out.println(ois.readUTF());
    • bject o = ois.readObject();//运行类型是Dog，抛出一个类找不到的异常

注意事项

  • 读写顺序需要一致
  • 要求序列化或者反序列的对象，需要实现Serilizable接口
  • 序列化的类中建议添加SerializableVersionUID,为了提高版本的兼容性
    • private static final long SerializableVersionUID
  • 序列化对象时候，默认对所有将所有属性都序列化，但除了static 或者 transient（不想序列化）修饰的成员
    • tostring输出后都是null
  • 被序列化的对象的属性如果也是对象，这个属性对象也需要实现序列化接口。
  • 序列化具备可继承性，如果某个类已经实现序列化，则它的所有子类也已经默认实现了序列化。
    • 比如Integer继承了Number，而Number实现了序列化接口

标准输入输出流

  • System.in 标准输入（System的一个属性） 类型 InputStream 键盘
    • 编译类型：InputStream
    • 运行类型：BufferedInputStream 包装流 字节流
  • System.out 标准输出 类型 PrintStream 显示器
    • 编译类型：PrintStream
    • 运行类型：PrintStream
  • new Scanner(System.in) 从键盘上读取
    • String next = scanner.next();

转换流 InputStreamReader OutStreamReader

  • 将字节流转换为字符流
  • 需求：文件乱码问题
    • ANSI国标码：根据系统决定。中国是GBK。
    • 编码不同，发现读取后乱码。
  • 当处理纯文本数据时，如果使用字符流效率更高，并且可以有效解决中文乱码问题。
  • 设置编码：utf-8 gbk gb2312

InputStreamReader

  • 继承了Reader
  • 构造器（inputStream Charset）可以把字节流 转成 字符流，并指定编码

OutStreamReader

  • 继承了Writer
  • 可以将字节流转换称为字符流。

配置文件Properties

  • 需求：有没有一个类能够让我很轻松地拿到左值和右边的值，而不需要用bufferreader在读取
  • properties介绍：
    • 是Hashtable的子类
    • 配置文件的格式：K-V。注意键值对不需要有空格，值不需要引号，默认是String。
  • properties常见方法
    • load：加载配置文件键值对到properties对象
    • list：将数据显示到指定设备/流对象
    • getProperty(key)：根据key获取value
    • setProperty(KEY,value)：设置键值对
    • store：将对象的键值对保存到文件。在IDEA中，如果含有中文，中文会被存储为unicode编码，而不是直接存储中午。
