数据库概述

基本概念

  • DB：database 数据库。
  • DBMS:database management system 数据库管理系统 ，如MySQL、Oracle
  • SQL：structured query language 结构化查询语言，标准的SQL支持所有数据库产品。不区分大小写。
    • 属于高级语言：执行的时候也会先编译然后再由DBMS执行。

什么是数据表？

  • 表，table是数据库的基本组成单元，所有的数据都以表格的形式组织，目的是可读性强。
  • 一个表内包含行和列
    • 行：也叫做数据、记录。
    • 列：也叫做字段。
  • 每一个字段的属性：字段名、数据类型、相关的约束

SQL语句的分类

  • DQL：数据查询语言。凡是select语句都是DQL。
  • DML：数据操作语言。对表中的数据进行增删改查。
  • DDL：数据定义语言：支持对表结构进行增删改查。
  • TCL：事务管理语言，支持提交事务和回滚事务。
  • DCL：数据控制语言：grant授权，revoke撤回权限等。

简单查询和条件查询

简单认识select

  • 语法格式：select 字段名 from 表名;
  • 字段可以进行数学运算。
    • select ename, sal*12 as 'yearsal' from empl; 查询sal将列重命名为年薪。（注意MySQL支持将字符串用双引号括起来，但是标准SQL要求用单引号，为了提高兼容性，建议不用双引号。）
  • 查询全部字段（实际开发不建议用）：select * from emp;

条件查询

  • 语法格式：select 字段 from 表名 where 条件;
  • 执行顺序：from — where — select
  • 例子：
    • 不等于：select ename,sal from emp where sal<>3000
    • 之间：
      • 第一种：select ename,sal from emp where sal>=1100 and sal<=3000
      • 第二种：select ename,sal from emp where sal between 1100 and 3000;
    • 字符串的之间：select ename from emp where ename between 'A' and 'C'
    • 注意：between and 在数字方面是闭区间，字符是左闭右开！！！
  • NULL
    • 在数据库中，NULL不是一个值，无法用等号衡量，必须用 is null 或者 is not null。
    • 例子：select ename,comm from emp where comm is null;
  • And和Or
    • and优先级高于or。
    • select ename,sal,depno from emp where sal>1000 and (deptno =20 or deptno = 30)
  • in等同于or
    • select ename,job from emp where job in(1000,5000) 这里不是区间而是具体的值
  • not in 不在值当中
    • select ename,job from emp where job not in(’salesman’,’manager’);
  • 模糊查询 like
    • %:任意多个字符
    • _:任意一个字符
    • \:转义符
    • 例如找名字最后一个字母是T. select ename from emp where ename like '%T'

排序

  • 语法: order by
  • 默认: 升序排序。指定排序：asc升序 desc 降序
  • 例：按照工资升序排序。
    • select ename,sal from emp order by sal
  • 多字段排序规则: 越靠前的字段越优先，只有当前面字段无法完成排序才会启用。
  • 例：按照工资降序排序，当工资相同按照名字升序排序。
    • select ename,sal from emp order by sal desc, ename asc;
  • 执行原理: from \ where \ select \ order
    • 证明： select ename,sal as salary from emp order by sal;

分组函数

引入和基本概念

  • 分组函数，也叫多行处理函数，所有的分组函数
  • 五个分组函数：count计数 \ sum 求和 \ avg 平均 \ max min
  • 特点：输入多行，输出一行。自动忽略NULL。
  • 例子：
    • 统计总记录条数 ： select count(*) from emp;
    • 统计非空记录条件； select count(field) from emp;
  • 单行处理函数
    • 特点：输入多少行，输出多少行。只要有一个NULL，最终为NULL
    • 解决：ifnull。 select ename,(sal + ifnull(comm,0)) from emp;
  • 引入group by：
    • 找出工资高于平均工资的员工。
      • 1111错误，无效使用分组函数。select ename,sal from emp where sal > avg(sal);
      • 原因：group by 在 where 后执行，而分组函数只有在group后执行。

group by 和 having

  • group by : 按照某个字段或者某些字段进行分组。
  • having : 对分组后某些数据进行再过滤。
  • 执行顺序 : from \ where \ group by \ having \ select \ order by
  • 分组函数一般和group by 联合使用，任何一个函数都是在group by 语句执行后才会执行。
  • 当一个语句中没有 group by ，整张表会自成一组。
  • 例子：
    • 找出每个工作岗位的最高薪资。select max(sal),job from emp group by job;
    • 找出工资高于平均工资的员工。select ename,sal from emp where select avg(sal) from emp;
      • 这里用到了子查询语句。
      • 第一步，找出平均工资 select avg(sal) from emp;
      • 第二步，按照平均工资分组 select ename,sal from emp where sal > 2073.214286
  • 当一条语句有 group by , select 语句只能跟参加分组的字段以及分组函数。
    • 比如这里ename无意义：select ename,max(sal),job from emp group by job;
  • 多个字段联合分组
    • 例如：找出每个部门的不同岗位的平均薪资
      • select deptno,job,max(sal) from emp group by deptno,job;
    • having的用法。例如：找出每个部门的最高薪资，要求显示薪资 > 2500的数据
      • select deptno,max(sal) from emp group by deptno having max(sal) > 2900
      • 缺点：效率低，计算多余了。
      • select deptno,max(sal) from emp where sal > 2900 group by deptno 利用where 先执行的特点，提高效率。
    • 结论：能用where就不用having。
    • where搞不定的情况？例如：找出每个部门的平均薪资，要求显示薪资大于2000的数据。
    • select deptno,avg(sal) from emp group by deptno having avg(sal) > 2000;

DQL多表连查

distinct: 查询结果集去重

  • 关键字：distinct 可以去除重复记录。只能出现在字段最前方。
    • 例如：select distinct job from emp;
  • 联合去重：要求两个条件都满足。
    • select distinct deptno,job from emp;
    • 例如：统计岗位的数量
      • select count(distinct job) from emp;

连接查询

什么是连接查询？

  • 实际开发中，通常将多张表联合起来查询。
  • 实际开发中，一个业务通常对应多张表。

连接查询的分类

  • 根据语法出现年代分类：
    • SQL92：一些老的DBA(Database administrator)可能还在使用。
    • SQL99：比较新的语法。
  • 根据表的连接方式来分类：
    • 内连接
      • 等值连接
      • 非等值连接
      • 自连接
    • 外连接
      • 左连接
      • 右连接
      • 全连接

连接查询原理和笛卡尔积现象

  • 引入：找出每一个员工的部门名称，要求显示员工名和部门名。
    • select ename,dname from emp,dept; 共56条记录
  • 笛卡尔积现象：当两张表进行连接查询的时候，如果没有任何条件限制，最终查询的结果条数 = 两表记录条数的乘积。
  • 表的别名的好处
    • 执行效率高
    • 可读性强
    • select e.ename d.dname from emp e , dept d;
  • 避免笛卡尔积现象：加条件过滤。
    • 避免了笛卡尔积现象，不会减少记录的匹配次数，次数还是56次，但是显示的是有效记录。
    • SQL92:select e.ename d.dname from emp e, dept d where e.deptno = d.deptno;

内连接之等值连接

  • 最大的特点：条件是等值关系。
  • 以查询每个员工的部门名字为例。
    • SQL92:select e.ename d.dname from emp e, dept d where e.deptno = d.deptno;
    • SQL99：
      • select e.ename d.dname
      • from emp e
      • inner join dept d
      • on e.deptno = d.deptn
    • SQL99语法: select xxx from A join B on 连接条件 where 过滤条件
    • 好处：表连接和连接后的过滤分开，不会都丢到where上。

内连接之非等值连接

  • 最大特点：连接条件是非等量关系，如between and
  • 案例：找出每个员工的工资等级，要求显示员工名和工资等级。
    • select e.ename,s.grade from emp e join salgrade s on e.sal between s.losal and s.hisal;

内连接之自连接

  • 最大特点：一张表看作两张表，自己连接自己。
  • 案例：找到每个员工的上级领导，要求员工名和对应领导名。
    • select a.ename,b.ename from emp a join emp b on a.mgr = b.empno;
    • king的上级领导是NULL，所以只显示13条记录。

外连接，实际开发中用的多

内外连接的区别

  • 内连接：A和B 进行连接，凡是A和B能匹配的记录都能查询，无主表和副表之分。
  • 外连接：A和B一张是主表，一张是副表。主要查询主表的内容，捎带副表的内容。当副表中没有数据和主表匹配，副表自动模拟出NULL与之匹配。
    • 左连接：左边那张表是主表。
    • 右连接：右边那张是主表。
  • 左连接有其对应写法，右连接有对应写法。
  • 例子：找出每个员工的上级领导，NULL也要显示。
    • select a.ename '员工' b.ename '上级' from emp a left join emp b on a.mgr = b.empno;
  • 例子：找出哪个部门没有员工
    • select d.* from emp e right join dept d on e.deptno = d.deptno where e.empno is null;

3表以上连接查询

  • 语法：A join B join C on xxx
  • 表示：A和B连接后，A再和C进行连接
  • 案例：找出每个员工的部门名称以及工资等级
    • select e.ename,d.dname,s.grade from emp e join dept
    • on e.deptno = d.deptno
    • join salgrade s
    • on e.sal between s.losal and s.hisal;
  • 案例：找出每个员工的部门名称 以及 工资等级 以及上级领导
    • select e.ename '员工',d.dname,s.grade e1 ename '领导' from emp e join dept d on e.deptno = d.deptno join salgrade s on e.sal between s.losal and s.hisal left join emp e1 on e.mgr = e1.empno;
