反射机制

引入

  • 需求：根据配置文件xx.properties 的信息，创建Cat对象并调用方法Hi
  • 问题：读取到的内容是字符串，所以传统方法无法做到。
  • 这样的需求，即通过外部文件配置，在不修改源码的情况下来控制程序，在学习框架的时候十分重要。同时，这样的设计也符合设计模式的OCP原则（开闭原则：在不修改源码的情况下，扩展程序的功能）。

反射机制

  • 反射机制允许程序在执行期间借助于ReflectionAPI取得任何类的内部信息（比如成员变量，构造器，成员方法等等），并能操作对象的属性和方法。
  • 加载完类之后，在堆中就产生了一个Class类型对象的对象（一个类只有一个Class对象），这个对象包含了类的完整结构信息。这个Class对象就像一面镜子，通过这个对象可以得到类的结构，所以形象称之为反射。
  • 反射机制原理图：代码编译阶段—-类加载器—->类加载阶段—->运行阶段

反射机制能做什么？

  1. 在运行时，判断任意一个对象所属的类。
  2. 在运行时，构造一个类的对象。
  3. 在运行时，得到任意一个类所具有的成员变量和方法。
  4. 在运行时，调用任意一个对象的成员变量和方法。
  5. 生成动态代理。（还不太清楚）

反射相关的类

  1. java.lang.Class 代表一个类，Class对象表示某个类加载后在堆区的对象。
  2. java.lang.reflect.Method 代表类的方法，Method对象表示某个类的方法
  3. java.lang.reflect.Field 代表类的成员，Field对象表示某个类的成员变量
  4. java.lang.reflect.Constructor 代表类的构造方法，Constructor对象表示某个类的构造器

反射的优点和缺点？

  1. 优点：动态创建和使用对象（也是框架底层的核心），使用灵活，没有反射机制，框架技术就失去了底层支撑。
  2. 缺点：使用反射基本是解释执行，对执行的速度又影响。

反射调用优化：关闭访问检查

  1. Method和Field、Constructor对象都有setAcessible()暴破方法 （都继承了AccessibleObject）
  2. setAcessible作用：启动和禁用访问安全检查的开关
    1. TRUE：反射对象在使用时候取消访问检查，提高反射效率。
    2. FALSE：反射的对象执行访问检查

换句话说，setAcessible就像是一个后门，可以访问private修饰的成员。

Class类

Class类基本介绍

  • 类图：继承了Object，实现了Serializable、Type、GenericDeclaration
  • Class也是类，因此也继承Object。
  • Class类不是New出来的，是系统创建的，通过类加载器（ClassLoader中的loadClass方法）。
    • 第一次运行new Cat() ，Debug会进入到loadClass方法。
    • 用forName方法也会进入loadClass
  • 对于某个类的Class类对象，在内存中只有一份，因为类只加载一次。
  • 每个对象实例都会记得自己是由哪一个Class实例所生成。
  • 通过Class对象可以完整地得到一个类的完整结构，通过一系列API
    • newInstance\getName\getSuperName\getInterface\getClassLoader
    • getConstructors\getMethod\getDeclaredFields\
    • …
  • Class对象是存放在堆区上的。
  • 类的字节码二进制数据，是放在方法区的，有的地方称之为类的元数据（包括方法代码、变量名、方法名、访问权限）
    • 方法区的二进制数据会引用到堆区的Class类对象

演示Class类的常用方法

如何获取Class类对象

  1. 前提：已知一个类的全类名，且该类在类路径下，可通过Class类的静态方法forName获取
    1. 可能抛出ClassNotFoundException。
    2. 实例：Class cls = Class.forName(“java.lang.Cat”);
    3. 应用场景：多用于配置文件，读取类全路径，加载类（底层框架用的多，必须在xml读）
  2. 前提：若已知具体的类，通过类的class获取，该方式最为安全可靠，程序性能最高。
    1. 实例：Class cls2 = Cat.class;
    2. 应用场景：多用于参数传递，比如通过反射得到对应构造器对象。
  3. 前提：已知某个类的实例，调用该实例的getClass方法
    1. 实例：Class clazz = 对象.getClass()
    2. 应用场景：通过创建好对象，获取Class对象
  4. 其它方式—类加载器 【JVM阶段深入学习】
    1. ClassLoader cl = 对象.getClass().getClassLoader();
    2. Class clazz4 = cl.loadClass(“类的全类名”);
  5. 基本数据：int char boolean float double byte long short 等按如下方式得到Class类
    1. Class cls = 基本数据类型.class
  6. 基本数据类型对应的包装类：可以通过.TYPE得到Class类
    1. Class cls = 包装类.TYPE

哪些类型有Class对象？

  1. 外部类，成员内部类，静态内部类，局部内部类，匿名内部类
    1. Class.class
  2. 接口interface
    1. Serializable.class
  3. 数组
    1. Integer[].class
    2. float[][].class
  4. 枚举enum
    1. Thread.State.class
  5. 注解annotation
    1. Deprecated.class
  6. 基本数据类型
  7. void
    1. void.class

类加载

类加载基本说明

反射机制是Java实现动态语言的关键，也就是通过反射实现类动态加载。

静态加载和动态加载

  • 静态加载：编译时加载相关的类，如果没有这个类则报错，依赖性太强。
  • 动态加载：运行时加载需要的类，如果运行时不用该类，则不报错，降低了依赖性。

类加载的时机

  • （静态）创建对象时候，new
  • （静态）当子类被加载的时候，父类也加载
  • （静态）调用类中的静态成员
  • （动态）通过反射

类加载的过程图

源码—字节码—类加载的阶段（加载、连接{验证、准备、解析}、初始化）

类加载的五个阶段

加载阶段Loading

JVM在该阶段的主要目的是将字节码从不同的数据源（可能是jar包、网络、class文件）转化为二进制字节流加载到内存中，并生成一个代表该类的java.lang.Class类对象

连接阶段—验证

  • 目的是确保Class文件的字节流中包含的信息符合当前虚拟机的要求，并且不会危害虚拟机自身的安全。
  • 包括：
    • 文件格式验证（是否以魔数 oxcafebabe开头）
    • 元数据验证
    • 字节码验证
    • 符号引用验证
  • 可以考虑使用：-Xverify:none 参数来关闭大部分的类验证措施，缩短虚拟机类加载的时间。

连接阶段—准备

  • JVM会在该阶段对静态变量分配内存并默认初始化（对应的数据类型的默认初始值，如0L，0，null，false等）。这些变量所使用的内存都将在方法区中进行分配。
    • 实例变量： public int n1 = 10; 在准备阶段不会分配内存。
    • 静态变量： public static int n2 = 20; 在准备阶段会初始化，但是是默认初始化，是0不是20
    • 常量变量： public static final int n3 = 30; 因为常量一次赋值就不改变，所以是初始化为30。

连接阶段—解析

  • 虚拟机将常量池内的符号引用替换为直接引用的过程。
    • 原先编译阶段没有内存地址，只是一种符号关系。

初始化阶段

  • 到初始化阶段，才真正执行类中定义的Java程序代码，此阶段是执行<clinit>()方法的过程
  • <clinit>()方法是由编译器按语句在源文件中出现的顺序，依次自动收集类中的所有静态变量的赋值动作和静态代码块中的语句，并进行合并。
    • 收集：类中静态代码块、静态变量赋值的语句；
    • 合并：例如两句赋值语句重复了，只输出一句。
  • 虚拟机会保证一个类的<clinit>()方法在多线程环境中被正确地加锁、同步，如果多个线程同时去初始化一个类，那么只会有一个线程去执行这个类中的<clinit>()，其它线程都需要阻塞等待，直到活动线程执行<clinit>()方法完毕。
    • 正因为有这个机制，才能保证某个类在内存中只有一份Class对象

反射获取类的结构信息

  • 常用的API如下：

  • Class类

  • Field类

  • Method类

  • Constructor类

  • Modifier是修饰符值，规定：

    • 默认是0，public是1，private是2，protected是4
    • static是8，final是16

通过反射创建对象

  • 方式1：调用类中public修饰的无参构造器
  • 方式2：调用类中指定的构造器
  • Class类相关的方法：
    • newInstance：调用类中的无参构造器，获取相应类的对象。
    • getConstructor(Class..clazz) 根据参数列表，获取对应的public构造器对象。
    • getDecalaredConstructor(Class…clazz) 根据参数列表，获取相应的所有的构造器对象。
  • Constructor类相关方法：
    • setAccessible 暴破：留了后门，可以访问私有构造器
    • newInstance(Object …obj)调用构造器

反射操作属性

  • 根据属性名获取Field对象：
    • Field f = clazz对象.getDeclaredField(属性名);
  • 暴破：
    • f.setAccessible(true);
  • 访问：
    • f.set(o,val);
    • syso(f.get(o));
  • 如果是静态属性，则set和get中的参数o，可以写成null。
    • 静态属性和类相关，而不是和对象相关。

反射暴破操作方法

  • 根据方法名和参数列表获取Method方法对象，
    • Method m = clazz.getDeclaredMethod(方法名，XXX.class);
  • 获取对象：
    • Object o = clazz.newInstance();
  • 暴破：
    • m.setAccessible(true);
  • 访问：
    • Object returnValue = m.invoke(o，实参列表);
  • 注意：如果是静态方法，则invoke的参数o可写成null。
